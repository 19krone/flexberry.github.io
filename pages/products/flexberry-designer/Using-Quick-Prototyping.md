Быстрая прототипизация удобна, когда требуется очень быстро создать и запустить систему.

Что генерируется при прототипизации:
# Генерирует класс со стереотипом `[Application](application.html)`
# Генерирует [L-](l-view.html) и [E-](e-view.html) представления для каждого класса данных
# Генерирует классы со стереотипом `[ListForm](Формы-списка-классы-со-стереотипом-listform.html)` для каждого класса данных
# Генерирует классы со стереотипом `[EditForm](Формы-редактирования-классы-со-стереотипом-editform.html)` для каждого класса данных
''__Примечание__: классом данных являются классы со стереотипом `[Классы-данных-классы-со-стереотипом-implementation-или-без-указания-стереотипа-и-их-своиства|implementation]` или без стереотипа.''

Для воспроизведения нижеприведённого примера требуется иметь установленный `MS SQL Server`.


Рассмотрим пример, который Вы самостоятельно, последовательно, по шагам можете воспроизвести, - создание информационной системы - телефонного справочника:
# Создайте новый проект, назовите его "ТелефонныйСправочник"; 
# Создайте в проекте последовательно конфигурацию, стадию, с любыми именами; 
# Создайте в стадии систему, назовите "Главная"; 
# Создайте в системе диаграмму классов, назовите "Структура"; 
# Поместите на диаграмму и свяжите между собой классы, как показано:![](/images/pages/img/Flexberry plugins/phonebook1.gif)и сохраните диаграмму. 
# Установите курсор на систему, где расположена диаграмма, затем выберите пункт меню "CSharp\Создать прототип":![](/images/pages/img/Flexberry plugins/startprototyping.jpg)
# Введите имя приложения и префикс в открывшемся окне и нажмите кнопку "ОК":![](/images/pages/img/Flexberry plugins/startprototyping1.jpg)
Рядом с диаграммой "Структура" (в той же системе) автоматически создастся диаграмма "ТелефонныйСправочник", которая содержит классы форм (для редактирования и для списка), а также класс-приложение:![](/images/pages/img/Flexberry plugins/phonebook2.gif)
Для каждого объекта создастся 2 представления: списковое и для редактирования; добавится описание (Description, Caption).
# Откройте свойства класса-приложения и измените в конфигурации параметры соединения с источником данных: имя сервера ('''SERVER=''') и базы данных ('''DATABASE=''', база данных необязательно должна существовать).

''__Примечание__: Адресом сервера является Ваш собственный компьютер ('''localhost'''). Если по каким-то причинам на рабочем месте не установлен MS SQL Server, необходимо обратиться к преподавателю.''![](/images/pages/img/Flexberry plugins/configprops.jpg)
# Отредактируйте в свойствах стадии [настройки для MS SQL Server direct generator ](configure--m-s--s-q-l--server-direct-generator.html) (укажите точно такую строку соединения с источником данных, как Вы указали в конфигурации приложения).![](/images/pages/img/Flexberry plugins/configbase.jpg)
# Отредактируйте в свойствах стадии [Настроики-генератора-кода-для-C-в-конкретнои-стадии|настройки для модуля CSharp ] (укажите, какое хотите, название компании и продукта, а также установите каталог генерации, как Вам удобно), например:![](/images/pages/img/Flexberry plugins/startprototyping2.jpg)
# Выберите пункт меню "MS SQL Server direct generator\Привести в соответствие...". Flexberry создаст базу данных и сгенерирует скрипты для создания структур данных, примените их.

''__Примечание__: если во время генерации происходит ошибка, необходимо
#* Проверить, выполнен ли пункт 9) (не указано ли в строке соединения SERVER=server;DATABASE=database - если указано, необходимо изменить в соответствии с действительностью)
#* Если все настроено верно, но ошибка продолжает возникать, необходимо обратиться к преподавателю или сотруднику технологического отдела''
